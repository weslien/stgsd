  <action>
Create the default status command that runs when `stclaude` is invoked with no subcommand.

**Create `spacetimeclaude/src/cli/commands/status.ts`:**

This module exports a function `registerStatusCommand(program: Command)` that sets the default action on the program.

Logic flow:
1. Get git remote URL via `getGitRemoteUrl()`
2. Connect to SpacetimeDB via `withConnection()`
3. Find project where `gitRemoteUrl` matches the local git remote
4. If no project found, call `outputError` with `PROJECT_NOT_FOUND`
5. Find project state where `projectId` matches the project's id
6. Find all phases where `projectId` matches the project's id
7. Return `{ project, state, phases }` data

Human-readable formatter (per locked decision -- "minimal one-liners"):
```
Project: {project.name}
Phase: {state.currentPhase} - {currentPhaseName} ({phaseStatus})
Plan: {state.currentPlan} of {totalPlansInPhase}
Last activity: {formattedDate} - {state.lastActivityDescription}
```

If no project state exists, show:
```
Project: {project.name}
Status: No state recorded yet
```

For the human formatter:
- Format the `lastActivity` timestamp: convert `microsSinceUnixEpoch` BigInt to a Date, then format as ISO date string (YYYY-MM-DD)
- Look up the current phase name from the phases array using `state.currentPhase` (which is a phase number string)

Error handling:
- Wrap the entire action in try/catch
- If error is a CliError, use `outputError(err.code, err.message, opts)`
- Otherwise, use `outputError(ErrorCodes.INTERNAL_ERROR, String(err), opts)`

IMPORTANT: After `outputSuccess` or `outputError`, call `process.exit(0)` to prevent Node.js from hanging due to SpacetimeDB SDK internal handles. `outputError` already calls `process.exit(1)`, so only add `process.exit(0)` after the success path.

NOTE on esbuild banner: The `--banner:js` flag value must be the shebang line. esbuild v0.24 syntax is `--banner:js='#!/usr/bin/env node'`. If the shell has quoting issues, try: `--banner:js="#!/usr/bin/env node"`.

NOTE on potential ESM issues: If the bundled output fails at runtime with `__dirname is not defined` or similar CJS-in-ESM errors, the fix is to add these shims in the esbuild banner:
```
#!/usr/bin/env node
import { fileURLToPath as __stclaude_furl } from 'node:url';
import { dirname as __stclaude_dir } from 'node:path';
const __filename = __stclaude_furl(import.meta.url);
const __dirname = __stclaude_dir(__filename);
```
But try without shims first -- the SpacetimeDB SDK may not need them.
