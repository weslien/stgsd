Two discrete fixes in this task:

**Fix 1: Update install:cli script in package.json**

Read `spacetimeclaude/package.json`. Change the `install:cli` script from:

```json
"install:cli": "bun run build:cli && mkdir -p ~/.claude/bin && cp dist/stclaude.mjs ~/.claude/bin/stclaude.mjs && chmod +x ~/.claude/bin/stclaude.mjs"
```

To:

```json
"install:cli": "bun run build:cli && mkdir -p ~/.claude/bin && cp dist/stclaude.mjs ~/.claude/bin/stclaude.mjs && chmod +x ~/.claude/bin/stclaude.mjs && ln -sf stclaude.mjs ~/.claude/bin/stclaude"
```

The `ln -sf` flags ensure:
- `-s`: Creates a symbolic link (not a hard link)
- `-f`: Replaces any existing file/symlink at the destination

The target `stclaude.mjs` is a RELATIVE path. This is critical -- do NOT use an absolute path like `/Users/gustav/.claude/bin/stclaude.mjs`. The symlink resolves relative to its directory (`~/.claude/bin/`).

Node.js v22 follows symlinks and detects the `.mjs` extension on the target file for ESM module type detection. This was verified during research.
