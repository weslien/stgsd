**Create `spacetimeclaude/src/cli/lib/project.ts`:**

This module provides three shared helper functions that every command uses. Import `DbConnection` type from module_bindings and `CliError`/`ErrorCodes` from errors.

```typescript
import type { DbConnection } from '../../module_bindings/index.js';
import { CliError, ErrorCodes } from './errors.js';
```

**`findProjectByGitRemote(conn, gitRemoteUrl)`:**
Iterate `conn.db.project.iter()` to find the project where `row.gitRemoteUrl === gitRemoteUrl`. Return the full project row. If not found, throw `new CliError(ErrorCodes.PROJECT_NOT_FOUND, ...)`.

**`findProjectState(conn, projectId)`:**
Iterate `conn.db.projectState.iter()` to find the row where `row.projectId === projectId`. Return the row or `null` if not found (project state may not exist yet for a fresh project).

**`findPhaseByNumber(conn, projectId, number)`:**
Iterate `conn.db.phase.iter()` to find the row where `row.projectId === projectId && row.number === number`. Return the full phase row. If not found, throw `new CliError(ErrorCodes.PHASE_NOT_FOUND, ...)`.

IMPORTANT: Phase numbers are strings (support decimals like "2.1"). Always compare as strings. The `number` parameter from CLI args is already a string from Commander.js.

IMPORTANT: All table iteration uses `conn.db.tableName.iter()` with manual filtering. Do NOT use `.filter()` on indexes from the client side -- those are server-side index accessors.

**Create `spacetimeclaude/src/cli/commands/get-state.ts`:**

Export `registerGetStateCommand(program: Command)` that registers a `get-state` subcommand.

The get-state command returns richer data than the default status command. It includes velocity data, session continuity, and plan counts -- everything an agent needs to understand project position.

Logic flow:
1. `getGitRemoteUrl()` to get project identity
2. `withConnection()` to connect and query
3. Inside the callback:
   - `findProjectByGitRemote(conn, gitRemoteUrl)` for project
   - `findProjectState(conn, project.id)` for state (may be null)
   - Iterate `conn.db.phase.iter()` filtered by `projectId === project.id` to collect all phases (include id, number, name, status, goal)
   - Iterate `conn.db.plan.iter()` to collect all plans for the project's phases (use a Set of phase IDs). Include id, phaseId, planNumber, status.
   - Return a structured object: `{ project: { id, name, gitRemoteUrl, coreValue }, state: { currentPhase, currentPlan, currentTask, lastActivity, lastActivityDescription, velocityData, sessionLast, sessionStoppedAt, sessionResumeFile } | null, phases: [...], plans: [...] }`

Human-readable formatter for get-state (richer than the default status):
```
Project: {project.name}
Core Value: {project.coreValue}

Phase: {state.currentPhase} - {phaseName} ({phaseStatus})
Plan: {state.currentPlan} of {plansInPhase}
Task: {state.currentTask}
Last activity: {formattedDate} - {state.lastActivityDescription}

Session: {state.sessionLast}
Stopped at: {state.sessionStoppedAt}

Phases:
  1. {name} ({status}) - {planCount} plans
  2. {name} ({status}) - {planCount} plans
  ...
```

If state is null, show:
```
Project: {project.name}
Status: No state recorded yet
```

Format the `lastActivity` timestamp: `Number(state.lastActivity.microsSinceUnixEpoch / 1000n)` -> `new Date(ms).toISOString().slice(0, 10)`.

After `outputSuccess()`, call `process.exit(0)`.

Error handling: same try/catch pattern as status.ts (CliError -> outputError with code, other -> outputError with INTERNAL_ERROR).

**Create `spacetimeclaude/src/cli/commands/get-phase.ts`:**

Export `registerGetPhaseCommand(program: Command)` that registers a `get-phase <number>` subcommand.

The `<number>` argument is a required string (Commander.js passes it as string). This is the phase number (e.g., "1", "2", "3", "2.1").

Logic flow:
1. `getGitRemoteUrl()` to get project identity
2. `withConnection()` to connect and query
3. Inside the callback:
   - `findProjectByGitRemote(conn, gitRemoteUrl)` for project
   - `findPhaseByNumber(conn, project.id, phaseNumber)` for the phase (throws PHASE_NOT_FOUND if missing)
   - Iterate `conn.db.plan.iter()` filtered by `phaseId === phase.id` to collect plans. Include id, planNumber, type, wave, objective, status, requirements, dependsOn, autonomous.
   - Iterate `conn.db.requirement.iter()` filtered by `projectId === project.id && phaseNumber === phase.number` to collect requirements. Include id, number (the req ID string like "CLI-01"), description, status, category.
   - Return: `{ phase: { id, number, name, slug, goal, status, dependsOn, successCriteria }, plans: [...], requirements: [...] }`

Human-readable formatter:
```
Phase {phase.number}: {phase.name}
Goal: {phase.goal}
Status: {phase.status}
Depends on: {phase.dependsOn || 'Nothing'}

Success Criteria:
{phase.successCriteria}

Plans ({plans.length}):
  {planNumber}. {objective} [{status}]
  ...

Requirements ({requirements.length}):
  {number}: {description} [{status}]
  ...
```

After `outputSuccess()`, call `process.exit(0)`.

Error handling: same pattern as get-state.
