Create the `read-plan <phase> <plan>` subcommand that returns the full content and metadata of a specific plan.

**Create `spacetimeclaude/src/cli/commands/read-plan.ts`:**

Export `registerReadPlanCommand(program: Command)`.

Register command: `program.command('read-plan <phase> <plan>').description('Read plan content and metadata')`.

Commander.js passes both `<phase>` and `<plan>` as strings. The `<phase>` is a phase number string (e.g., "3"). The `<plan>` is a plan number (e.g., "1", "2") that needs to be converted to BigInt via `BigInt(planArg)`.

Logic flow:
1. `getGitRemoteUrl()` for project identity
2. `withConnection()` to connect and query
3. Inside the callback:
   - `findProjectByGitRemote(conn, gitRemoteUrl)` for project
   - `findPhaseByNumber(conn, project.id, phaseArg)` for the phase (throws PHASE_NOT_FOUND)
   - Parse `planArg` to BigInt: wrap in try/catch, if invalid throw `new CliError(ErrorCodes.INVALID_ARGUMENT, 'Plan number must be a positive integer')`
   - Iterate `conn.db.plan.iter()` to find the plan where `row.phaseId === phase.id && row.planNumber === planNumberBigInt`. If not found, throw `new CliError(ErrorCodes.PLAN_NOT_FOUND, ...)`.
   - Iterate `conn.db.planTask.iter()` filtered by `row.planId === plan.id` to collect tasks. Include id, taskNumber, type, description, status, commitHash.
   - Iterate `conn.db.mustHave.iter()` filtered by `row.planId === plan.id` to collect must-haves. Include id, truths, artifacts, keyLinks.
   - Return structured object: `{ phase: { number, name }, plan: { id, planNumber, type, wave, dependsOn, objective, autonomous, requirements, status, content }, tasks: [...], mustHaves: [...] }`

Human-readable formatter:
```
Phase {phase.number}: {phase.name}
Plan {plan.planNumber}: {plan.objective}
Type: {plan.type} | Wave: {plan.wave} | Status: {plan.status}
Autonomous: {plan.autonomous}
Requirements: {plan.requirements}
Depends on: {plan.dependsOn || 'Nothing'}

Content:
{plan.content}

Tasks ({tasks.length}):
  {taskNumber}. [{status}] {description}
  ...

Must-Haves:
  Truths: {mustHave.truths}
  Artifacts: {mustHave.artifacts}
  Key Links: {mustHave.keyLinks}
```

If the plan has no content (empty string), show `Content: (empty)`.

After `outputSuccess()`, call `process.exit(0)`.

Error handling: same try/catch pattern as other commands.

IMPORTANT: BigInt comparison -- use `=== planNumberBigInt` (both BigInt), not `=== Number(planArg)`. CLI arg is string, convert to BigInt first.

**Create `spacetimeclaude/src/cli/commands/roadmap.ts`:**

Export `registerRoadmapCommand(program: Command)`.

Use Commander.js nested subcommand pattern:
```typescript
const roadmapCmd = program
  .command('roadmap')
  .description('Roadmap operations');
roadmapCmd
  .command('analyze')
  .description('Analyze roadmap status')
  .action(async () => { /* ... */ });
```

The `analyze` subcommand returns a structured overview of all phases with plan and requirement completion counts. This lets agents verify project progress and check consistency between stored status and actual plan completion.

Logic flow:
1. `getGitRemoteUrl()` for project identity
2. `withConnection()` to connect and query
3. Inside the callback:
   - `findProjectByGitRemote(conn, gitRemoteUrl)` for project
   - Iterate `conn.db.phase.iter()` filtered by `projectId === project.id` to collect all phases. For each phase, collect: id, number, name, status, goal, dependsOn.
   - Sort phases by number: use `parseFloat(a.number) - parseFloat(b.number)` for numeric ordering (handles "2.1" between "2" and "3").
   - Iterate `conn.db.plan.iter()` to build a map of phaseId -> plans array. For each plan, track: planNumber, status.
   - Iterate `conn.db.requirement.iter()` filtered by `projectId === project.id` to build a map of phaseNumber -> requirements array. For each requirement, track: number (the req ID), status.
   - For each phase, compute:
     - `plans.total`: number of plans in this phase
     - `plans.completed`: number of plans with status "complete" or "completed"
     - `requirements.total`: number of requirements mapped to this phase
     - `requirements.completed`: number of requirements with status "complete" or "completed" or "done"
   - Return: `{ project: { name, coreValue }, phases: [{ number, name, status, goal, dependsOn, plans: { total, completed }, requirements: { total, completed } }] }`

NOTE on status matching: Use case-insensitive matching for completion status checks. Normalize by converting to lowercase and checking if it starts with "complete" or equals "done". This handles variations like "Complete", "Completed", "complete".

Human-readable formatter:
```
Roadmap: {project.name}
{project.coreValue}

Phases:
  {number}. {name}
     Status: {status}
     Plans: {plans.completed}/{plans.total} complete
     Requirements: {requirements.completed}/{requirements.total} complete
  ...

Overall: {completedPhases}/{totalPhases} phases complete
```

After `outputSuccess()`, call `process.exit(0)`.

Error handling: same pattern.
