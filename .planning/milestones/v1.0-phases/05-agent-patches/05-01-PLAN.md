waitForInsert helper pattern (define locally in each write command file):
```typescript
function waitForInsert(
  conn: DbConnection,
  table: { onInsert: (cb: (_ctx: any, row: any) => void) => void },
  matchFn: (row: any) => boolean,
  timeoutMs = 5000,
): Promise<void> {
  return new Promise((resolve, reject) => {
    const timer = setTimeout(() => {
      reject(new CliError(ErrorCodes.INTERNAL_ERROR, `Insert confirmation timed out after ${timeoutMs / 1000} seconds`));
    }, timeoutMs);
    table.onInsert((_ctx: any, row: any) => {
      if (matchFn(row)) {
        clearTimeout(timer);
        resolve();
      }
    });
  });
}
```

```typescript
program
  .command('write-plan')
  .description('Persist a plan to SpacetimeDB')
  .requiredOption('--phase <phase>', 'Phase number')
  .requiredOption('--plan <plan>', 'Plan number within phase')
  .requiredOption('--objective <text>', 'Plan objective')
  .option('--type <type>', 'Plan type (execute/tdd)', 'execute')
  .option('--wave <wave>', 'Execution wave number', '1')
  .option('--depends-on <text>', 'Plan dependencies (comma-separated plan IDs)', '')
  .option('--autonomous', 'Plan is fully autonomous (no checkpoints)', true)
  .option('--no-autonomous', 'Plan has checkpoints')
  .option('--requirements <text>', 'Requirement IDs (comma-separated)', '')
  .option('--status <status>', 'Plan status', 'pending')
  .option('--content <text>', 'Full plan content (markdown)')
  .option('--content-file <path>', 'Read plan content from file instead of --content')
  .option('--tasks-json <json>', 'Tasks as JSON array: [{taskNumber, type, description}]', '[]')
  .option('--must-haves-json <json>', 'Must-haves as JSON: {truths, artifacts, keyLinks}')
  .action(async (options) => { /* ... */ });
```

Logic inside the action:
1. `getGitRemoteUrl()` for project identity
2. If `--content-file` provided, read the file content using `fs.readFileSync(options.contentFile, 'utf-8')`. This allows the planner to write plan content to a temp file and pass the path, avoiding shell escaping issues with large markdown content.
3. Validate `--tasks-json` and `--must-haves-json` are valid JSON (try/catch JSON.parse)
4. Inside `withConnection()`:
   a. Resolve project: `findProjectByGitRemote(conn, gitRemoteUrl)`
   b. Resolve phase: `findPhaseByNumber(conn, project.id, options.phase)`
   c. Set up `waitForInsert(conn, conn.db.plan, (row) => row.phaseId === phase.id && row.planNumber === BigInt(options.plan))` BEFORE calling reducer
   d. Call `insertPlan` reducer:
      ```typescript
      conn.reducers.insertPlan({
        phaseId: phase.id,
        planNumber: BigInt(options.plan),
        type: options.type,
        wave: BigInt(options.wave),
        dependsOn: options.dependsOn,
        objective: options.objective,
        autonomous: options.autonomous,
        requirements: options.requirements,
        status: options.status,
        content: planContent,  // from --content or --content-file
      });
      ```
   e. Await insert confirmation
   f. Find the inserted plan to get its ID (iterate `conn.db.plan.iter()`, find row matching phaseId + planNumber)
   g. If `--tasks-json` provided and non-empty array, insert each task:
      ```typescript
      const tasks = JSON.parse(options.tasksJson);
      for (const task of tasks) {
        conn.reducers.insertPlanTask({
          planId: insertedPlan.id,
          taskNumber: BigInt(task.taskNumber),
          type: task.type || 'auto',
          description: task.description,
          status: task.status || 'pending',
          commitHash: '',
        });
      }
      ```
      Wait briefly (1s) for task inserts to propagate.
   h. If `--must-haves-json` provided, insert must-have:
      ```typescript
      const mh = JSON.parse(options.mustHavesJson);
      conn.reducers.insertMustHave({
        planId: insertedPlan.id,
        truths: typeof mh.truths === 'string' ? mh.truths : JSON.stringify(mh.truths),
        artifacts: typeof mh.artifacts === 'string' ? mh.artifacts : JSON.stringify(mh.artifacts),
        keyLinks: typeof mh.keyLinks === 'string' ? mh.keyLinks : JSON.stringify(mh.keyLinks),
      });
      ```
   i. Return `{ phaseNumber: options.phase, planNumber: options.plan, objective: options.objective, taskCount: tasks.length }`

Import `fs` from node for `--content-file` support: `import { readFileSync } from 'node:fs';`

Export `registerCompletePhaseCommand(program: Command)`.

This command is called by execute-phase.md (PTCH-04) after verification passes, to mark a phase done and advance project state. It replaces `gsd-tools.cjs phase complete`.

```typescript
program
  .command('complete-phase <phase>')
  .description('Mark a phase as complete and advance project state')
  .action(async (phaseArg: string) => { /* ... */ });
```

Logic:
1. `getGitRemoteUrl()` for project identity
2. Inside `withConnection()`:
   a. Resolve project: `findProjectByGitRemote(conn, gitRemoteUrl)`
   b. Resolve phase: `findPhaseByNumber(conn, project.id, phaseArg)`
   c. Get current state: `findProjectState(conn, project.id)`

   d. Update phase status to "Complete":
      ```typescript
      conn.reducers.updatePhase({
        phaseId: phase.id,
        number: phase.number,
        name: phase.name,
        slug: phase.slug,
        goal: phase.goal,
        status: 'Complete',
        dependsOn: phase.dependsOn,
        successCriteria: phase.successCriteria,
      });
      ```

   e. Determine the next phase: iterate `conn.db.phase.iter()`, filter by `projectId === project.id`, sort by `parseFloat(number)`, find the first phase after the completed one. If none exists, this is the last phase.

   f. Set up `waitForStateUpdate(conn, project.id)` BEFORE calling upsertProjectState
   g. Update project state to point to next phase:
      ```typescript
      conn.reducers.upsertProjectState({
        projectId: project.id,
        currentPhase: nextPhase ? nextPhase.number : state.currentPhase,
        currentPlan: nextPhase ? 1n : state.currentPlan,
        currentTask: 0n,
        lastActivityDescription: `Completed phase ${phase.number}: ${phase.name}`,
        velocityData: state.velocityData,
        sessionLast: new Date().toISOString().split('T')[0],
        sessionStoppedAt: `Completed phase ${phase.number}`,
        sessionResumeFile: state.sessionResumeFile ?? '',
      });
      ```
   h. Await state update confirmation
   i. Return `{ completedPhase: { number: phase.number, name: phase.name }, nextPhase: nextPhase ? { number: nextPhase.number, name: nextPhase.name } : null, isLastPhase: !nextPhase }`

```
  Next: {nextPhase.number} - {nextPhase.name} (or "Last phase - milestone complete")
```
