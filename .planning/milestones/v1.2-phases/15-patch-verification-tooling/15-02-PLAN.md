Key implementation details:

**resolveManifestPath()**: Check `~/.claude/stclaude/patch-manifest.json` first. If not found, try walking up from the CLI's location to find the repo root `patch-manifest.json` (for dev mode). Throw MANIFEST_NOT_FOUND if neither exists.

**resolveGsdDir()**: Check `process.cwd()/.claude/get-shit-done/VERSION`, then `~/.claude/get-shit-done/VERSION`. Return the directory that has a VERSION file. If neither exists, return null (files in this category will be reported as "missing").

**Main logic:**
- Read and parse the manifest JSON
- For each category (gsdPaths, agentPaths, commandPaths), resolve the base directory
- For each file entry, construct the full path, read the file content
- Check each expected pattern with `content.includes(pattern)` — collect missing ones
- Check each forbidden pattern with `content.includes(pattern)` — collect present ones
- If file doesn't exist on disk, mark as "missing" status
- Collect all results into an array

**Human formatter:**
- Print header "Patch Verification Results" with "=" underline
- For each result: `{relative_path}  {PASS|FAIL|MISSING}` (right-aligned status)
- For failures: indent and show `MISSING: {pattern}` and `FORBIDDEN PRESENT: {pattern}` lines
- Summary line: `Results: X/Y passed, Z failed`
- If any failures: `\nRun /gsd:reapply-patches to restore missing patches.`

**Exit behavior:**
- All pass: outputSuccess with results, exit 0 (normal)
- Any fail: outputSuccess with results (so the full table is shown), then process.exit(1)
- Manifest not found: outputError with MANIFEST_NOT_FOUND
