<objective>
Create the patch-manifest.json file that catalogs all expected and forbidden patterns across every GSD file patched by stclaude.

Purpose: This manifest is the source of truth for the verify-patches command. Without it, verification has nothing to check against.

Output: A single JSON file at the repo root with per-file expected/forbidden pattern arrays for all ~33 patched files.
</objective>

<context>
The manifest must cover three categories of files, each with different path resolution:

1. **GSD paths** (relative to `~/.claude/get-shit-done/`): 25 workflow files + 1 reference + 1 template
2. **Agent paths** (relative to `~/.claude/agents/`): 5 agent files (gsd-executor.md, gsd-planner.md, gsd-verifier.md, gsd-phase-researcher.md, gsd-codebase-mapper.md)
3. **Command paths** (relative to `~/.claude/commands/`): 1 command file (gsd/debug.md)

The manifest structure:
```json
{
  "version": "1.0.0",
  "description": "Expected/forbidden patterns for stclaude-patched GSD files",
  "gsdPaths": {
    "workflows/progress.md": {
      "expected": ["stclaude init progress", "stclaude get-state"],
      "forbidden": ["cat .planning/STATE.md"]
    }
  },
  "agentPaths": {
    "gsd-executor.md": {
      "expected": ["stclaude init execute-phase"],
      "forbidden": []
    }
  },
  "commandPaths": {
    "gsd/debug.md": {
      "expected": ["stclaude get-debug"],
      "forbidden": []
    }
  }
}
```

**Pattern selection guidelines:**
- Pick 2-5 distinctive stclaude command invocations per file as expected patterns
- For files with many references, pick the most important/distinctive ones (the command names that represent key functionality)
- For forbidden patterns, only include `.planning/` or `gsd-tools.cjs` references that have been replaced. If a file still legitimately uses `.planning/` for unreplaced features, do NOT add those as forbidden.
- Use literal string matching (not regex) — patterns should be exact substrings

**To determine correct patterns:** Read each file, find the stclaude references, and select the most representative 2-5. For forbidden patterns, check if any old `.planning/` file reads or `gsd-tools.cjs` calls remain that should have been replaced.
