**Subcommand: `init progress`** (CLI-03)

Assembles context for the progress workflow. Returns a JSON bundle with:

```typescript
interface InitProgressData {
  project: { id: bigint; name: string; coreValue: string; description: string };
  state: {
    currentPhase: string;
    currentPlan: bigint;
    currentTask: bigint;
    lastActivity: { microsSinceUnixEpoch: bigint };
    lastActivityDescription: string;
    velocityData: string;
    sessionLast: string;
    sessionStoppedAt: string;
    sessionResumeFile: string | undefined;
  } | null;
  phases: Array<{
    number: string;
    name: string;
    status: string;
    goal: string;
    dependsOn: string;
    plans: { total: number; completed: number };
    requirements: { total: number; completed: number };
  }>;
  recentSummaries: Array<{
    planId: bigint;
    phaseNumber: string;
    headline: string;
  }>;
}
```

Logic inside `withConnection`:
1. `findProjectByGitRemote(conn, gitRemoteUrl)` for project
2. `findProjectState(conn, project.id)` for state (may be null)
3. Collect all phases for this project (iterate `conn.db.phase.iter()`, filter by `projectId === project.id`), sort by `parseFloat(number)`
4. Build phase-to-plans map (iterate `conn.db.plan.iter()`, group by phaseId)
5. Build phaseNumber-to-requirements map (iterate `conn.db.requirement.iter()`, filter by projectId)
6. For each phase, compute plan/requirement counts using `isCompletionStatus()` (case-insensitive status check: `lower.startsWith('complete') || lower === 'done'`)
7. Collect recent summaries: iterate `conn.db.planSummary.iter()`, join with plans to find the phase number. Return the 5 most recent (by comparing planId descending as a rough proxy for recency since auto-inc IDs are monotonically increasing, though not strictly sequential).
8. Return the assembled `InitProgressData` object

Human-readable formatter: Show project name, state summary, phase list with completion counts, and recent summary headlines.

```typescript
program
  .command('seed')
  .description('Bootstrap a new project in SpacetimeDB from current git repo')
  .requiredOption('--name <name>', 'Project name')
  .requiredOption('--description <text>', 'Project description')
  .requiredOption('--core-value <text>', 'Core value statement')
  .option('--constraints <text>', 'Project constraints', '')

  .option('--key-decisions <text>', 'Key decisions', '')
  .option('--phases-json <json>', 'Phases as JSON array string', '[]')
  .option('--requirements-json <json>', 'Requirements as JSON array string', '[]')
  .action(async (options) => { /* ... */ });
```

Logic inside the action:
1. `getGitRemoteUrl()` for git remote URL
2. Inside `withConnection()`:
   a. **Check if project already exists:** Try to find project by git remote URL. Unlike other commands that throw on not-found, here we CATCH the CliError(PROJECT_NOT_FOUND). If project exists, throw a user-friendly error: `new CliError(ErrorCodes.INVALID_ARGUMENT, 'Project already exists for this git remote. Use update commands instead.')`. If the error is PROJECT_NOT_FOUND, that means no project exists -- proceed.
   b. Set up an `onInsert` listener on `conn.db.project` to confirm the seed completed:
      ```typescript
      const insertPromise = new Promise<void>((resolve, reject) => {
        const timer = setTimeout(() => {
          reject(new CliError(ErrorCodes.INTERNAL_ERROR, 'Seed confirmation timed out after 10 seconds'));
        }, 10_000);
        conn.db.project.onInsert((_ctx, row) => {
          if (row.gitRemoteUrl === gitRemoteUrl) {
            clearTimeout(timer);
            resolve();
          }
        });
      });
      ```
   c. Call the reducer:
      ```typescript
      conn.reducers.seedProject({
        gitRemoteUrl,
        name: options.name,
        description: options.description,
        coreValue: options.coreValue,
        constraints: options.constraints,
        context: options.context,
        keyDecisions: options.keyDecisions,
        phasesJson: options.phasesJson,
        requirementsJson: options.requirementsJson,
      });
      ```
   d. Await `insertPromise`
   e. After confirmation, read back the created project:
      ```typescript
      const project = findProjectByGitRemote(conn, gitRemoteUrl);
      ```
   f. Return `{ project: { id: project.id, name: project.name, gitRemoteUrl: project.gitRemoteUrl } }`

Use a 10-second timeout for seed (larger than the 5s default) since seed_project does bulk inserts.
