Create two mutation commands that modify project state via the `upsertProjectState` reducer.

CRITICAL PATTERN for all mutation commands:
1. Read current state first (to preserve unchanged fields)
2. Call the reducer with ALL fields (existing + changed)
3. Wait for the onUpdate/onInsert callback to confirm the write propagated
4. Return the confirmed new state
5. Timeout after 5 seconds if confirmation never arrives

**Helper function (define in each command file or in a shared util):**

Create a `waitForStateUpdate` helper. This can be defined locally in each file since it is small:

```typescript
function waitForStateUpdate(
  conn: DbConnection,
  projectId: bigint,
  timeoutMs = 5000,
): Promise<void> {
  return new Promise((resolve, reject) => {
    const timer = setTimeout(() => {
      reject(new CliError(ErrorCodes.INTERNAL_ERROR, 'State update timed out after 5 seconds'));
    }, timeoutMs);

    const done = () => {
      clearTimeout(timer);
      resolve();
    };

    conn.db.projectState.onUpdate((_ctx, _oldRow, newRow) => {
      if (newRow.projectId === projectId) done();
    });
    conn.db.projectState.onInsert((_ctx, newRow) => {
      if (newRow.projectId === projectId) done();
    });
  });
}
```

Rather than duplicating this in each file, create it once in `advance-plan.ts` and also in `update-progress.ts` and `record-metric.ts`. The duplication is acceptable for 3 small command files -- extracting to a shared module is optional.

**Create `spacetimeclaude/src/cli/commands/advance-plan.ts`:**

Export `registerAdvancePlanCommand(program: Command)`.

Register command: `program.command('advance-plan').description('Advance to the next plan in the current phase')`.

No arguments needed. The command reads current state and increments `currentPlan` by 1.

Logic flow:
1. `getGitRemoteUrl()` for project identity
2. `withConnection()` to connect and query (the callback returns a Promise)
3. Inside the callback:
   - `findProjectByGitRemote(conn, gitRemoteUrl)` for project
   - `findProjectState(conn, project.id)` for current state
   - If state is null, throw `new CliError(ErrorCodes.INTERNAL_ERROR, 'No project state exists. Run a seed_project reducer first.')`
   - Compute new plan: `state.currentPlan + 1n`
   - Set up the `waitForStateUpdate(conn, project.id)` promise BEFORE calling the reducer
   - Call reducer:
     ```typescript
     conn.reducers.upsertProjectState({
       projectId: project.id,
       currentPhase: state.currentPhase,
       currentPlan: state.currentPlan + 1n,
       currentTask: 0n,  // reset task to 0 when advancing plan
       lastActivityDescription: `Advanced to plan ${state.currentPlan + 1n}`,
       velocityData: state.velocityData,
       sessionLast: state.sessionLast,
       sessionStoppedAt: state.sessionStoppedAt,
       sessionResumeFile: state.sessionResumeFile ?? '',
     });
     ```
   - Await the `waitForStateUpdate` promise
   - After confirmation, read the updated state from the cache:
     ```typescript
     const updatedState = findProjectState(conn, project.id);
     ```
   - Return `{ project: { id: project.id, name: project.name }, previousPlan: state.currentPlan, newPlan: state.currentPlan + 1n, state: updatedState }`

IMPORTANT: Set up the onUpdate/onInsert listener BEFORE calling the reducer. Otherwise you might miss the event if the round-trip is faster than the listener setup.

IMPORTANT: `sessionResumeFile` may be `undefined` on the current state row. The reducer param is `t.string()` (not optional), so pass `state.sessionResumeFile ?? ''` to convert undefined to empty string.

Human-readable formatter:
```
Advanced plan: {previousPlan} -> {newPlan}
Phase: {state.currentPhase}
```

Helper function for parsing velocity data:
```typescript
function parseVelocityData(raw: string): { plans: Array<{ id: string; duration: number; phase?: string; recordedAt?: string }> } {
  if (!raw || raw.trim() === '') return { plans: [] };
  try {
    const parsed = JSON.parse(raw);
    if (parsed && Array.isArray(parsed.plans)) return parsed;
    return { plans: [] };
  } catch {
    return { plans: [] };
  }
}
```
